**Feature Branch**: `005-cloudevents-alignment`  
**Date**: 2026-03-29

## Entities

### Signal Envelope (Jido.Signal struct — used for all PubSub messages)

Every PubSub broadcast uses this uniform format. No raw tuples.

| Field | Type | Required | Description |
|-------|------|----------|-------------|
| `id` | `String.t()` | Yes | UUID7 unique identifier |
| `type` | `String.t()` | Yes | Event classification (e.g., `"task.created"`) |
| `source` | `String.t()` | Yes | Event producer URI (e.g., `"/jido_tasks/tools/add_task"`) |
| `subject` | `String.t()` | No | Entity URI the event relates to |
| `time` | `String.t()` | Auto | ISO 8601 timestamp |
| `specversion` | `String.t()` | Auto | `"1.0.2"` |
| `data` | `term()` | No | Event payload |

### Subject URI Patterns

| Context | Pattern | Example |
|---------|---------|---------|
| Agent-scoped | `/agents/#{agent_id}` | `/agents/sess_abc123` |
| Task in workspace | `/workspaces/#{wid}/tasks/#{tid}` | `/workspaces/ws_1/tasks/42` |
| Agent in workspace | `/workspaces/#{wid}/agents/#{sid}` | `/workspaces/ws_1/agents/sess_abc` |

## Migration Map: Tuple → Signal

| Before (tuple) | After (signal type) | Subject |
|-----------------|--------------------|---------| 
| `{:artifact_update, sid, name, data, mode}` | `"artifact.#{name}"` | `/agents/#{sid}` |
| `{:agent_signal, sid, signal}` | `signal.type` (passthrough) | `/agents/#{sid}` |
| `{:message_completed, sid, response}` | `"murmur.message.completed"` | `/workspaces/#{wid}/agents/#{sid}` |
| `{:new_message, sid, msg}` | `"murmur.message.received"` | `/workspaces/#{wid}/agents/#{sid}` |
| `{:task_created, task}` | `"task.created"` | `/workspaces/#{wid}/tasks/#{tid}` |
| `{:task_updated, task}` | `"task.updated"` | `/workspaces/#{wid}/tasks/#{tid}` |

## Validation Rules

- Every PubSub broadcast MUST be a `%Jido.Signal{}` struct
- `type` field MUST follow dot-separated convention: `{domain}.{entity}.{action}`
- `source` field MUST be a URI path starting with `/`
- `subject` field SHOULD be a URI path when entity context is available
- `id` MUST be a UUID7 (auto-generated by `Signal.new!/3`)
