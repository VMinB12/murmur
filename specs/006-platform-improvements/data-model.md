# Data Model: Platform Infrastructure Improvements

**Feature Branch**: `006-platform-improvements`  
**Date**: 2026-03-29

## Entities

### Topic Helper Module (`JidoMurmur.Topics`)

Centralized PubSub topic construction. All topics follow `workspace:{wid}:...` hierarchy.

| Function | Return Format | Description |
|----------|--------------|-------------|
| `agent_artifacts(wid, sid)` | `"workspace:#{wid}:agent:#{sid}:artifacts"` | Artifact updates for an agent session |
| `agent_stream(wid, sid)` | `"workspace:#{wid}:agent:#{sid}:stream"` | LLM streaming for an agent session |
| `agent_messages(wid, sid)` | `"workspace:#{wid}:agent:#{sid}:messages"` | Messages to/from an agent session |
| `workspace_tasks(wid)` | `"workspace:#{wid}:tasks"` | Task events in a workspace |
| `workspace(wid)` | `"workspace:#{wid}"` | Workspace-level events |

**Migration from old topics**:

| Old Format | New Function |
|-----------|-------------|
| `"agent_artifacts:#{sid}"` | `agent_artifacts(wid, sid)` |
| `"agent_stream:#{sid}"` | `agent_stream(wid, sid)` |
| `"workspace:#{wid}:agent:#{sid}"` | `agent_messages(wid, sid)` |
| `"jido_tasks:tasks:#{wid}"` | `workspace_tasks(wid)` |

### Config Validator (`JidoMurmur.Config`)

Startup validation for required configuration keys.

**Required keys per package**:

| Package | Key | Type | Description |
|---------|-----|------|-------------|
| `:jido_murmur` | `:repo` | module | Ecto repo module |
| `:jido_murmur` | `:pubsub` | module | Phoenix.PubSub module |
| `:jido_murmur` | `:jido_mod` | module | Jido integration module |
| `:jido_murmur` | `:otp_app` | atom | Application name |
| `:jido_tasks` | `:repo` | module | Ecto repo module |
| `:jido_tasks` | `:pubsub` | module | Phoenix.PubSub module |

**Error format**:
```
** (RuntimeError) Missing required configuration for :jido_murmur

  The following keys are not configured:
    - :repo
    - :pubsub

  Add to your config/config.exs:

    config :jido_murmur,
      repo: MyApp.Repo,
      pubsub: MyApp.PubSub

  Or run: mix jido_murmur.install
```

### Agent Profile Behaviour (`JidoMurmur.AgentProfile`)

| Callback | Return Type | Description |
|----------|------------|-------------|
| `name/0` | `String.t()` | Agent display name |
| `description/0` | `String.t()` | Agent description |
| `system_prompt/0` | `String.t()` | System prompt text |
| `tools/0` | `[module()]` | List of available Jido.Action modules |
| `plugins/0` | `[module()]` | List of Jido.Plugin modules |
| `opts/0` | `keyword()` | Additional agent options |

All callbacks are already generated by `use Jido.AI.Agent`. Adding `@behaviour JidoMurmur.AgentProfile` to profile modules enables compile-time enforcement.

## Validation Rules

- All PubSub topics MUST be constructed via `JidoMurmur.Topics` functions
- No inline topic string interpolation permitted
- Config validation runs once at supervisor init — not per-request
- Telemetry events follow `:telemetry.span/3` convention (start/stop/exception)
- Profile modules MUST implement all behaviour callbacks or receive compiler warnings
